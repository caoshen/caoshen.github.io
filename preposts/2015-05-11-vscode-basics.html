visual studio code 基础

发表于 2015-05-11   |   分类于 ， ， ， ， ，   |  

visual studio code 是一个代码编辑器。 和其他编辑器一样， vscode 拥有常见的用户界面： 左边是资源管理器， 显示用到的所有的文件和文件夹； 右边是编辑区， 显示所有打开文件的内容。

除此以外， vscode 还有许多独特的特性。

文件、 文件夹和项目

vscode 是基于文件和文件夹的， 你可以从打开一个文件或者文件夹开始。

除此以外， vscode 可以读取和利用定义在不同的框架和平台下的项目文件。 例如， 如果你打开包含一个或者多个 package.json， project.json， tsconfig.json 或者 ASP.Net v5 Visual Studio 方案和项目文件， vscode 会读取它们并利用它们来提供额外的功能比如编辑器智能感知（IntelliSense）。

启动 VSCode

Vscode 可以从命令行启动来快速打开文件、 文件夹或者项目。

打开文件 index.html

打开文件夹 c:\src\Contents

vscode 并不区分打开的是文件夹或者项目。 如果打开的文件夹包含项目文件， vscode 会读取这些文件并在状态栏显示它们的上下文。 如果有多个项目， 从状态栏可以切换项目。

从文件夹 c:\src\WebApp 文件夹中打开项目

换句话说， 打开包含项目的文件夹就打开了项目

打开当前文件夹

基础布局

vscode 的简单布局使得你可以最大化编辑区同时又有足够的空间来浏览文件或者项目。 它的界面分为四个部分：

  1. 编辑区 编辑文件的主要区域， 最多可以打开三个边靠边的编辑区
  2. 侧边栏 包含像资源管理器的视图来帮助你， 当你在项目中工作时
  3. 状态栏 指示打开的项目信息或者文件信息
  4. 视图栏 在最左边， 让你可以在视图之间切换， 并且给你额外的指示信息， 比如激活 Git 后外部变化的数目。

每次你打开 vscode 后， 它会以你上次退出时保持的状态来打开。 文件夹、 布局和打开的文件得以保留。

vscode 允许最多3个可见编辑器使得你可以把3个文件边靠边编辑， 而不是把它们放在不同的标签页中。

这样减少了管理标签页所需的时间花费， 但不会限制工作的文件数目。 资源管理器视图使得你可以快速找到你需要的文件。

你可以把侧边栏移到右边（ View 菜单 - Move Sidebar ） 或者切换它的可见性（Ctrl + B）

分栏编辑

使用 vscode 最多可以分三栏编辑

如果编辑器已经打开，有多重方式可以分栏打开：

  • 在资源管理器中按住 Ctrl (Mac: Cmd) 点击文件
  • Ctrl+\ 把当前文件分成两栏
  • 在资源管理器中文件的右键菜单上选择 Open to the Side

当你打开其他文件时， 被激活的编辑器会显示那个文件的内容。 所以如果你有两个编辑器， 你想在右边的编辑器打开’foo.cs’文件， 确保右边的编辑器是活动的（ 需要点击一下） 。

你可以调整编辑器的大小和顺序。 拖动编辑器的标题区来移动它们。

资源管理器

资源管理器用来浏览， 打开和管理所有项目中的文件和文件夹。

在 vscode 中打开文件夹之后， 文件夹的内容会显示在资源管理器中， 你可以：

  • 创建， 删除， 重命名文件和文件夹
  • 通过拖动来移动文件和文件夹
  • 使用右键菜单的选项

你可以把 vscode 外部的文件拖进资源管理器

在资源管理器的顶部是一个叫做 Working Files 的区域。 它是一些活动的文件， 你之前在 vscode 中打开的文件会显示在里面。 举个例子， 如果你

  • 改变了一个文件
  • 在资源管理器中双击了文件
  • 打开了不属于当前文件夹的文件

那么这个文件会在工作文件部分。

如果你完成了工作， 可以使用 Close All 来关闭所有工作文件

使用 Ctrl + E 来快速搜索和打开文件

vscode 可以和其他你可能使用的工具一同使用， 特别是命令行工具。 如果你想要在文件夹中打开命令行工具， 右击文件夹然后选择 Open in Console。

如果你想在系统资源管理器中打开文件或者文件夹， 右键选择 Reveal in Explorer (或者 Reveal in Finder Mac)

保存/自动保存

在默认情况下， vscode 需要显式地保存你的更改(Ctrl+S)。

但是， 可以使用 Auto Save 在空闲时保存你的更改。 当选了自动保存后， 不再需要显示保存。 如果你在使用文件监视器来自动化工作， 你可能需要关闭自动保存来确保当任务在后台运行时， 你有更大的控制权。

可以使用 Ctrl + Shift + P 来打开控制面板， 输入 auto。

当然也可以在 File 菜单中选择自动保存。

搜索

vscode 可以使用 Ctrl + Shift + F 来搜索。 所有的结果以文件和行的形式显示出来。 打开文件查看出现的位置， 然后点击出现位置在编辑器中查看。

也可以用正则表达式搜索

可以在搜索栏中按 Ctrl + Shift + J 来开启高级搜索选项。

vscode 默认排除了一些文件夹来减少你不感兴趣的搜索结果（ 比如： node_modules） 。 打开设置来改变 search.excludeFolders 部分的规则。

可以在管理器中右键点击文件夹来选择 Find in Folders 在文件夹中搜索

控制面板

vscode 也可以通过键盘来操作。 最重要的快捷键是 Ctrl + SHift + P， 可以打开控制面板。 从这里可以使用 vscode 的所有功能， 包括一些常用操作的快捷键。

操作面板提供了许多命令， 你可以执行编辑命令， 打开文件， 搜索符号， 使用交互窗口查看文件大纲。 举些例子：

  • Ctrl + E 通过输入名字将你导航到任意文件或者符号
  • Ctrl + Tab 将显示后一个打开的文件（切换文件）
  • Ctrl + Shift + P 显示控制面板中的编辑命令
  • Ctrl + Shift + O 将你导向文件中的特殊符号
  • Ctrl + G 跳到文件某一行

文件导航

vscode 提供了快速方便的文件导航。 使用 Ctrl + Tab 查看所有打开的文件。

也可以使用 Alt + Left/Right 在文件之间切换

使用 Ctrl + E 打开任意文件
