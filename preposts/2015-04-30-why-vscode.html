为什么使用 Visual Studio Code

Visual Studio Code (vsc) 为开发者提供了一种新的开发工具。 vsc 将简洁和”编码-编辑-调试”循环的流水线特点结合在代码编辑器中。 vsc 是 Visual Studio 家族中的第一个代码编辑器和第一个跨平台开发工具-同时支持 OSX， Linux 和 Windows 操作系统。

vsc 是一个适合日常使用的强大快速的代码编辑器。 虽然现在是预发布版本， 它仍然具有开发者需要的一些特性: 导航， 自定义快捷键， 语法高亮， 括号匹配， 自动缩进， 代码段， 并且支持多种程序语言。

除此以外， vsc 还提供了自带的代码补全， 语义理解， 导航， 重构功能。 vsc 使用与 visual studio 相同的技术， 支持 C# 的 ASP.NET 5 开发和 TypeScript 和 JavaScript 的 Node.js 开发。 vsc 也为 web 技术如 HTML， CSS， LESS， SASS 和 JSON 提供了大量工具。 vsc 同时集成了包管理器， 仓库， 构建和其他常用任务来使得日常工作流变得更快。 vsc 也集成了 Git， 可以使用 git 操作和源代码 diff 操作。

开发者并不将他们的时间都花费在写代码: 因为他们还需要调试程序。 调试是 vsc 中的另一个常用特性， 也是具有精简编码体验的IDE所具有的一个特性。 vsc 具有流水线型的集成的调试体验。 目前它支持 Node.js 调试， 以后会添加更多的调试。

在结构上， vsc 结合了 web， native和特定语言技术中最好的部分。 使用GitHub Electron Shell， vsc 将 web 技术如 JavaScript 和 Node.js 和本地应用的速度和灵活性相结合。 vsc 使用了相同的基于 HTML 的工业编辑器的更新更快的版本， 这个编辑器驱动了“ Manaco” 云编辑器、 Internet Explorer的 F12 工具和其他项目。 vsc 使用了工具服务结构使得他可以使用驱动 visual studio 的相同技术， 包括 Roslyn for .Net ， TypeScript 和 visual studio 调试引擎等。 在以后， 我们会改进和精简这个结构， vsc 会包括一个公共可扩展模型， 使得开发者构建和使用插件， 更多地自定义”编码-编辑-调试” 体验。

当然， 现在 vsc 还处在早期版本。 如果你喜欢用代码编辑器开发或是开发跨平台的 web 和云应用， 你应该试试 vsc ， 并将你的体验告诉我们。
